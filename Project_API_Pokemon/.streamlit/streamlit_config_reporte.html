📘 Configuración de Streamlit: config.toml

Este documento explica detalladamente cómo configurar el archivo config.toml de Streamlit, qué hace cada instrucción, sus valores posibles y cómo afecta el comportamiento del servidor.

📁 Ubicación del archivo

El archivo config.toml debe ubicarse dentro de una carpeta oculta llamada .streamlit en la raíz del proyecto:

/mi-proyecto/.streamlit/config.toml

🔧 Sección [server]

1. headless = true

Ejecuta Streamlit en modo headless, es decir, sin abrir automáticamente el navegador al iniciar.

  • true: No abre navegador (modo silencioso).
  • false: Abre navegador automáticamente.

2. port = 8501

Define el puerto en el que se levantará el servidor de Streamlit.

  • Valor por defecto: 8501
  • Puedes cambiarlo a otro (ej. 8051, 8080) si el puerto está ocupado.

3. enableCORS = false

Controla si se permite el acceso desde otros dominios (Cross-Origin Resource Sharing).

  • true: Permite acceso desde otros orígenes.
  • false: Restringe acceso solo al mismo dominio.

4. enableXsrfProtection = false

Activa o desactiva la protección contra ataques Cross-Site Request Forgery (XSRF).

  • true: Activa protección con tokens de seguridad.
  • false: Desactiva protección (útil en desarrollo).

📄 Ejemplo completo de configuración

[server]
headless = true
port = 8051
enableCORS = false
enableXsrfProtection = true

Este ejemplo configura el servidor para ejecutarse en modo silencioso en el puerto 8051, sin permitir acceso externo y con protección CSRF activada.
